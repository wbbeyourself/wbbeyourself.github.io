OpenNMT运行环境搭建

前言：深度学习入门第一步就是搭建运行环境，安装各种框架，设置各种环境变量，其实熟练之后也就那么回事。下面我以OpenNMT为例，介绍搭建服务器运行环境的整个过程，供大家参考。

1、当前配置

  • 服务器：ubuntu 18.04 LTS

  • cuda: 9.0

  • cudnn: 7.3.0

  • conda: 4.5.11

查看这些版本的命令，请自行百度吧，这里就不赘述了。

2、OpenNMT文档

  • OpenNMT首页
  • OpenNMT Github
  • OpenNMT doc

3、环境配置

创建虚拟环境，这里我们使用 Python 3.7，环境名为opennmt。如果报错，就是镜像挂了，文末有解决方案。

conda create -n opennmt python=3.7

安装Pytorch 1.1

先进入虚拟环境，再用pip安装torch和torchvision。注：配置pip豆瓣源，速度非常快。

source activate opennmt
pip install torch torchvision

clone NMT，并安装依赖包。

git clone https://github.com/OpenNMT/OpenNMT-py.git
cd OpenNMT-py/
pip install -r requirements.txt

运行demo

python preprocess.py -train_src data/src-train.txt -train_tgt data/tgt-train.txt -valid_src data/src-val.txt -valid_tgt data/tgt-val.txt -save_data data/demo

遇到的问题

1、执行 conda create -n opennmt python=3.7 时，报错 Solving environment: failed

CondaHTTPError: HTTP 404 NOT FOUND for url https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/noarch/repodata.json
Elapsed: 00:00.316530

The remote server could not find the noarch directory for the
requested channel with url: https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free

解决方案：因为清华大学和中科大已经宣布停止提供 Anaconda 镜像服务。还是用回官方的吧。

命令步骤：

cd ~
rm .condarc
conda config --add channels  https://repo.anaconda.com/pkgs/free/
conda config --set show_channel_urls yes
