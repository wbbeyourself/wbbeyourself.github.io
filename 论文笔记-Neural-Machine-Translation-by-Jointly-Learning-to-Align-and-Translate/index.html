[论文笔记] Neural Machine Translation by Jointly Learning to Align and Translate

2019-07-08 925 字
Seq2Seq NMT NMT注意力机制

简介

这篇是 不来梅雅各布大学（Jacobs University Bremen）的 Dzmitry Bahdanau 发表于 ICLR 2015 的最早的关于在基于Encoder-Decoder框架中使用注意力机制（Attention）的论文。

论文地址：https://arxiv.org/pdf/1409.0473.pdf

背景

2014 年 发表于EMNLP 的《Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation》 首次将Encoder-Decoder框架用于NMT，并提出了GRU模型。该论文提出，将源语言的句子（词语序列），经过Encoder端的RNN编码成一个固定长度的向量（context vector），相当于将整个句子的语义压缩到一个固定长度大小的向量里，这种机制使得长句子被压缩后信息丢失，导致随着句子长度的增加，模型的性能急剧下降。

公式细节

1、Soft Attention 之 context vector的计算
$$
c_{i}=\sum_{j=1}^{T_{x} } \alpha_{i j} h_{j}
$$
每decode一个 $y_i$，都要计算一个$c_i$，其是$(h_1, h_2, …, h_T)$的线性加权之和，$T$表示源语言句子长度。这个公式中的每个$\alpha_{ij}$的计算也很关键，${\alpha { {\rm{ij} } } } \in [0,1],\sum\limits{j = 1}^T { {\alpha {ij} } } = 1$。其计算公式如下（本质上 就是一个softmax归一化嘛）：
$$
\alpha{i j}=\frac{\exp \left(e_{i j}\right)}{\sum_{k=1}^{T_{x} } \exp \left(e_{i k}\right)}
$$
其中每个$e_{ij}$的计算如下，很简单就是一个线性连接层（输入节点是$s_{i-1}$和$h_j$的拼接），再加一个$tanh$的激活函数，$s_{i-1}$是decoder上一个状态的隐藏状态，$h_j$是encoder在第$j$步的隐藏状态：
$$
e_{ij}=a\left(s_{i-1}, h_{j}\right)=v_{a}^{\top} \tanh \left(W_{a} s_{i-1}+U_{a} h_{j}\right)
$$

2、context vector 在decoder中的使用

先上一个GRU的图：

在本文中为了和encoder的$h_i$区分开来，decoder端的隐藏状态用符号$s_i$表示，此外decoder的输入符号也变了，分别为$s_{i-1}$和$y_t$，同时又加入了context vector，即符号$c_i$。

一句话总结，decoder端的GRU有三个输入，分别是$s_{i-1}$，$y_t$，$c_i$，输出为$s_i$。值得注意的是，标准的GRU只有两个输入，加入$c_i$相是这篇论文的一个创新点，这是典型的模型创新。

隐藏状态$s_i$的计算，与标准的GRU一模一样：
$$
s_{i}=f\left(s_{i-1}, y_{i-1}, c_{i}\right)=\left(1-z_{i}\right) \circ s_{i-1}+z_{i} \circ \tilde{s}{i}
$$
不同之处在于，所有在激活函数之前的运算都有$c_i$的身影，从下面就能看出来：
$$
\tilde{s}{i} =\tanh \left(W E y_{i-1}+U\left[r_{i} \circ s_{i-1}\right]+C c_{i}\right) \
z_{i} =\sigma\left(W_{z} E y_{i-1}+U_{z} s_{i-1}+C_{z} c_{i}\right) \
r_{i} =\sigma\left(W_{r} E y_{i-1}+U_{r} s_{i-1}+C_{r} c_{i}\right)
$$

相比于标准的GRU，在每个激活函数的括号里，都对$c_i$进行了一个线性连接运算。像$W$，$U$，$C$都是weights参数，$E y_{i-1}$是词语$y_{i-1}$经过embedding后的词向量。这样理解起来是不是简单多了。$c_i$的计算前面已经讲过了。

论文里就给了这么一个简单的图：
